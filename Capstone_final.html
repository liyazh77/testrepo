1. Report Summary¶

1.1 Business Problem¶

Gene mutation classification is an important problem in Precision Medicine to identify mutations that contribute to tumors growth (drivers) from neutral mutations (passengers). Once the most important mutation is identified, it becomes easier to determine the cancer cell and find drugs to treat them. At present, clinical pathologist manually reviews biomedical scholarly articles to classify genetic mutation one by one. This is a quite time-consuming task. However, once the most important mutation is identified, it's easier to bring new drugs to the market and find the appropriate drug to treat cancer. It will make a huge difference in patients' life. In this research, we aim to speed up the classification process with high performance. We first assess if clinical text can be used to classify genetic mutations. Then we used machine learning algorithms to automate classification of genetic mutations using text from biomedical scholarly articles. Some related research in this field include work by Zhang et.al (loglos=0.61, March 2019) and Singhal et.al (f1-score=0.845, July 2016).

1.2 Data Description¶

The dataset is sourced from a Kaggle competition (https://www.kaggle.com/c/msk-redefining-cancer-treatment/data). The competition is sponsored by Memorial Sloan Kettering Cancer Center (MSKCC) and has 3,321 labelled entries. Each ID, for which clinical evidence is provided as an unstructured text, is labelled as one of the 9 type of genetic mutations. The dataset is provided into two separate files. The first file provides information on clinical evidence for each ID. The second file provides genetic mutation, gene information, and aminoacid variation (for a mutation) corresponding to each ID.

  • Converted all text column to string (some are float type); to ensure that vectorized string functions can be applied for faster processing
  • Converted all letters in the text to lower case
  • Removed special characters using regex
  • Removed single digit numbers (appearing as words) using regex
  • Removed white spaces and single letter words using regex
  • Removed stopwords from text
  • Tokenized documents into words
  • Lemmatize words using function from nltk library
  • Combined all text related to a mutation type into a single string; this is further used by FreqDist and Text function in nltk library
  • Converted label to 9 dummy variables using str.dummy(); datatype of the column is converted to string before building dummy variables
  • One-hot encoding of words in a text for word embedding
  • Converting un-equal list to numpy array using pad_sequences function from keras library

1.5 Exploratory Analysis¶

Statistic summary of text and characters¶

  • On average, each clinical text has ~6,365 words. There are a few observations for which clinical evidence is not provided (cell is empty). We have dropped such rows as they are not expected to add value to the overall prediction. It was found that column separator in the data files is '|'

  • The number of characters in a document varies from 3 to a maximum of ~0.4 million with mean ~50,000. And the distribution is smilar with number of text words. We assume some of the documents with very little text should be excluded from the prediction as they may not have enough information for prediction.

Gene VS Genetic mutation class¶

  • Genetic mutation class 7 is the most dominant class, accounting for ~29% of the total. Some of the smaller mutation types are under-represented. Representation of class is especially low compared to other mutation classes. We expect this skewness could also impact prediction especially for categories that are under-represented.

  • There are 269 different types of genes in the dataset corresponding to 9 genetic mutation types. And we are able to get top 10 genes with mutations and identify which class is related to this gene. It is also found that some genes have strong relationship with the genetic mutation type for instance PTEN is dominated by class 4. This makes us believe it may be a good predictor for genetic mutation type.

  • Each gene is dominated by a particular mutation, indicating that gene is expected to be a strong predictor for genetic mutation type. There are around 3,309 different types of aminoacid change (variations) in the dataset. Out of these, 3,251 have only single observation associated with it. As a result, we have excluded this from prediction as most categories in this variable are not expected to have enough data for statistical significance.

Word impact¶

  • The wordcloud of top word shows the most common words in the text corpus of the literature; the top words appear to be the most common words associated with any discussion on cancer cell mutation and doesn't provide more about words that may be relevant to describe a mutation type. We need to dive deeper into the text to identify more details and words that may be related to a mutation type.

  • Go deeper from wordcloud, we get word corpus related to a gene class. From the top frequency plot, we can see the clear difference in the words association with cancel cell mutation class. This makes us believe that word token from clinical text may prove to be good predictor for gene mutation.

  • We found greater insights when words are looked into context with its surrounding words. For instance, for genetic mutation of class 2, we can see that egfr (EGFR mutation refers to a mutation (damage) to the portion of the DNA in a lung cancer cell) appears to be a growth factor receptor. This indicated that a model that includes contextual information of a word is expected to be a good model compared to bag-of-word (BoW) prediction models.

  • Based on above 3 findings, we believe that word embedding that captures semantic relationship among words will be a suitable modeling technique. And we applied what we found to the predictive model.

1.7 Conclusion¶

We have found that there are many words which are unique to genetic mutation type such as 'egfr, gefitinib words for genetic mutation type belonging to class 7. We also found that adding context to words (using nearby words) adds significant benefit to associate words with genetic mutation type. For instance, for genetic mutation of class 2, we can see that egfr (EGFR mutation refers to a mutation (damage) to the portion of the DNA in a lung cancer cell) appear regularly with words such as growth, factor, and receptor. This indicates that a model that includes contextual information of a word is expected to be a good model compared to bag-of-word (BoW) prediction models.

As a result, we believe that genetic mutation is predictable using clinical evidence text. We have developed 2 word embedding models and found a maximum accuracy of 58.80% and a f1-score of 0.5800 compared to a baseline accuracy of ~29%. We believe that both accuracy and f1-score can be further improved. Therefore, there is a need to extend this research and build more robust predictive model. To improve prediction accuracy or f1-score, we should try deeper and better (LSTM, Gensim etc.) neural networks to further improve predictions. We can consider including gene data to build more robust input for neural networks. We should also look to consider some prominent bigrams (as found in sections 4.8) to further improve the quality of predictors for the model. It was also observed that there are limited number of observations for some mutations. If we can include more observations for those mutations, we believe more improvement in the prediction can be obtained.

def pre_precess_text(Data_Frame=df,Target_Column_Name='Text', New_Column_Text='Edited_Text'):
    '''The function creates a new column in the dataframe with cleaned text; 
    cleaning tasks includes making all column containing text as string 
    (just in case the format is something else), letters to lower case, 
    removing special characters, removing single digit numbers, 
    removing white spaces, single letter words, and stopwords, converting words to root words using lemmatization'''
    
    #Convert each column in the dataframe to a string
    X=[]
    for i in range(0,len(Data_Frame)):
        X.append(str(Data_Frame.iloc[i][Target_Column_Name]))
    Data_Frame[New_Column_Text]=X
    #Convert Text to lower character
    Data_Frame[New_Column_Text]=Data_Frame[New_Column_Text].str.lower()
    #Remove special characters
    Data_Frame[New_Column_Text]=Data_Frame[New_Column_Text].str.replace('[!@#$%^&*()[]{};:\,/<>?\|`~-=_]','')
    #Remove single digit numbers
    Data_Frame[New_Column_Text]=Data_Frame[New_Column_Text].str.replace('\s[0-9]\s',' ')
    #Remove white spaces and single letter words
    remove_extra=[]
    s=string.ascii_lowercase
    for i in range(0,len(s)):
        remove_extra.append(s[i])
    for i in range(0,len(remove_extra)):
        a=' '+s[i]+' '
        Data_Frame[New_Column_Text]=Data_Frame[New_Column_Text].str.replace(a,' ')
    #Remove stopwords from the text and also some unnecessary words
    stop_words = list(set(stopwords.words('english')))
    z=len(stop_words)
    for s in stop_words[0:z]:
        a=' '+s+' '
        Data_Frame[New_Column_Text]=Data_Frame[New_Column_Text].str.replace(a,' ')
    Data_Frame=Data_Frame.reset_index()
    #Convert words in the text to root words using lemmatization
    del Data_Frame['index']
    for i in range(0,len(Data_Frame)):
        s=Data_Frame.iloc[i][New_Column_Text]
        # Regular expression tokenization
        words = regexp_tokenize(s, '[^. ]+')
        # Lemmatizataion
        lmtzr = WordNetLemmatizer()
        words = [lmtzr.lemmatize(word) for word in words]
        s=' '.join(words)
        Data_Frame.loc[i,New_Column_Text]=s
    return Data_Frame

#Function to get word corpus of all text related to a gene mutation class and process it with Lemmatization
def combine_text(Data_Frame=df,Column_Name='Edited_Text',Class=1):
    '''The function creates word tokens after combining all clinical text related to a class of genetic mutation'''
    
    #Subset the dataframe
    dt=df[df['Class']==Class].iloc[1:20,]
    #Select column with text as a panda series from dataframe and convert it into a list
    s=dt[Column_Name].tolist()
    #Join elements of the above list to prepare a single string
    s=' '.join(s)
    # Regular expression tokenization
    words = regexp_tokenize(s, '[^. ]+')
    return words

As a result, we believe that word embedding that captures semantic relationship among words will be a suitable modeling technique.

#Function to identify top bigrams
def ngrams_df(Data_Frame=df,Text_Column_Name='Edited_Text', n=20, Class=1, num_grams=3):
    '''The function creates a dataframe of ngrams (m consecutive words) for a specified class; 
    only the top n ngrams are returned'''
    
    #Subset dataframe so that new dataframe has only text related to relevant class
    New_Data_Frame=Data_Frame[Data_Frame['Class']==Class]
    #Convert panda series to list
    clinical_text=New_Data_Frame[Text_Column_Name].tolist()
    #tokenize clinical text in the list above
    clinical_text=[regexp_tokenize(text, '[^. ]+') for text in clinical_text]
    # Create list of lists containing bigrams in clinical text
    terms_ngram = [list(ngrams(text,num_grams)) for text in clinical_text]
    # Flatten list of ngrams in clean text
    ngram_new = list(itertools.chain(*terms_ngram))
    # Create counter of words in clean bigrams
    ngram_counts = collections.Counter(ngram_new)
    ngram_df = pd.DataFrame(ngram_counts.most_common(n),columns=['ngram', 'count'])
    return ngram_df
